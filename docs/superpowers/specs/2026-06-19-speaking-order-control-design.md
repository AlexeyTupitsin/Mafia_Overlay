# Контроль порядка давания слова

Дата: 2026-06-19

## Цель

Помочь ведущему вести дневной круг обсуждения: система отслеживает, кто уже
говорил, и подсказывает/не даёт нарушить три правила. Слово по-прежнему
выдаётся вручную кликом «🎙 Слово» — никакой автоматической передачи.

## Область

- Контроль только в **панели ведущего** (`public/control`). Оверлей трансляции
  не меняется.
- «Круг» = дневное обсуждение одного раунда: `phase.kind === 'day'` и
  голосование не активно (`voting.active === false`). Речи кандидатов во время
  голосования правилами круга не затрагиваются.

## Состояние

Новое поле в `createInitialState()` (`server/store.js`):

```js
speaking: {
  round: 1,            // дневной раунд, к которому относится текущий круг
  spoken: [],          // места в порядке выдачи слова в этом круге
  circleStarter: null, // место, открывшее текущий круг (первое слово)
  prevStarter: null    // место, открывшее прошлый день — основа подсказки старта
}
```

Состояние ведёт **сервер** (источник истины). Старые автосейвы без поля
`speaking` нормализуются при загрузке (подстановка дефолта).

## Правила

### 1. Один игрок — одно слово за круг

- Сервер: при `SET_SPEAKER` на место, которого ещё нет в `spoken` (и при условии
  «день + не голосование»), добавляет его в `spoken`; если `spoken` был пуст —
  это и есть `circleStarter`.
- UI: кнопка «🎙 Слово» у мест из `spoken` — `disabled` с тултипом «уже говорил
  в этом круге».
- Клик по текущему спикеру (выключение слова, `seat: null`) не блокируется и из
  `spoken` ничего не удаляет.

### 2. Молчанка при 3 фолах

- Игрок с `fouls === state.settings.foulSilenceAt` (по умолчанию 3) не может
  получить слово **до конца текущего круга**.
- UI: кнопка `disabled` с тултипом «молчанка (3 фола)».
- В новом круге право восстанавливается автоматически (молчанка отыграна), пока
  фолов не стало `foulRemoveAt` (4 → удаление, и так недоступен).

### 3. Подсказка стартового места (мягко)

- Пока `spoken` пуст, в панели подсвечивается рекомендованное стартовое место:
  следующий живой по кругу после `prevStarter`.
- День 1 (или `prevStarter === null`): первое живое место, считая от №1.
- Это **только подсветка** — слово можно выдать кому угодно, блокировки нет.

## Сброс круга

При переходе в новый дневной раунд (`SET_PHASE` на `day` с новым `round`,
`NEXT_PHASE` ночь→день):

```
speaking.prevStarter   = speaking.circleStarter  // кто открывал закрывшийся день
speaking.spoken        = []
speaking.circleStarter = null
speaking.round         = новый round
```

`prevStarter` обновляется значением `circleStarter` только если тот был задан
(в дне реально кто-то говорил); иначе сохраняется прежний `prevStarter`.

## Жёсткость защиты

Защита — **только в UI** (disabled-кнопки). Сервер `SET_SPEAKER` не отклоняет
повтор/молчащего: он лишь ведёт состояние круга. Это сохраняет за ведущим
техническую возможность обхода и упрощает обработку краёв.

## Затрагиваемые файлы

- `server/store.js` — поле `speaking` в initial state + нормализация автосейва.
- `server/actions.js` — `SET_SPEAKER` ведёт `spoken`/`circleStarter`;
  `SET_PHASE`/`NEXT_PHASE` сбрасывают круг.
- `public/control/control.js` — disabled-кнопки по правилам 1–2, бейдж
  «говорил», подсветка рекомендованного старта (правило 3).
- `tests/` — юнит-тесты на ведение `spoken`/`circleStarter`/`prevStarter` и
  расчёт рекомендованного старта.

## Не входит (YAGNI)

- Автоматическая кнопка «Следующий».
- Серверные отказы/валидация по правилам.
- Отражение круга на оверлее трансляции.
- Жёсткое принуждение стартового места.
